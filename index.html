🚪📦 Tokseg - MVP

❗ Descrição do Problema

O recebimento de encomendas em portarias remotas enfrenta desafios como falta de segurança, dependência de funcionários e riscos de perdas ou extravios.

💡 Descrição da Solução

Criar um sistema de segurança para portarias e escritórios que aumente a confiabilidade, reduza extravios e facilite o acesso sem depender de funcionários internos.

🗺️ Jornada do Usuário

  1. ✨ Descoberta e primeira interação: Recebe a encomenda, descobre e tenta usar o armário.
  2. 🔄 Uso contínuo: Começa a usar regularmente e percebe as vantagens.
  3. 🛠️ Problemas e suporte: Enfrenta e soluciona problemas com ajuda do sistema.
  4. 📢 Primeira utilização: Recomenda o serviço aos vizinhos.

📝 Backlog do Produto

  • 📩 Notificações e alertas via e-mail.
  • 📊 Histórico e relatórios de uso.

🛠️ Stack de Desenvolvimento

  • 💻 Linguagem: Java
  • ⚙️ Framework: Spring Boot
  • 🗄️ Banco de Dados: PostgreSQL
  • 🔑 Autenticação: JWT
  • 🖥️ IDE: Eclipse
  • 🌐 Hospedagem: GitHub

🔄 Lógica de Funcionamento

  1. 👤 Cadastro e configuração: Morador e gestor configuram o sistema.
  2. 📦 Entrega: Entregador deposita a encomenda com liberação automatizada.
  3. 📲 Notificação e retirada: Morador é notificado e retira a encomenda.
  4. 🛡️ Gestão: Admin monitora, gera relatórios e gerencia exceções.
